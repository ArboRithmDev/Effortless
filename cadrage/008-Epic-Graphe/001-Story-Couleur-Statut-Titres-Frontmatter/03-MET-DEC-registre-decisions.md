## 🏛️ Decision List

### 1. `DEC-01` : Amélioration graphe : livrable code + configs plugin documentées (côté app)
* **Status** : Accepted
* **Date** : 2026-07-01
* **Phase** : O-analyse
* **Context** : EVO-014 visait un graphe plus lisible (couleur statut, noms lisibles, relations). Or Obsidian possède les fichiers de config des plugins et écrase toute édition externe à chaud : impossible de fiabiliser front-matter-title / extended-graph par du code sur un vault ouvert.
* **Decision** : Livrable CODE de l'Epic Graphe : colorGroups par statut dans graph.json (template + live) et champ titre dans le frontmatter des docs story (générateur cadrage_frontmatter + backfill_titre de l'existant). Les réglages plugin restants (Dataview, front-matter-title clé=titre+graphe, extended-graph couleur/relations) sont côté application (une fois par l'utilisateur) et documentés dans cadrage/9-Guide-Obsidian.md, shippé au template.
* **Consequences** :
  - Nouveaux vaults : graphe colorisé + titre prêts ; guide 9-Guide-Obsidian.md embarqué pour les 3 réglages app.
  - Noms de nœuds lisibles dès que front-matter-title pointe sur titre (réglage app une fois).
  - Axe relations (extended-graph) traité comme option documnentée, pas comme code (Obsidian possède sa config).
  - titre ajouté à 106 docs + tous les futurs ; visible aussi dans Properties/Dataview/Bases.
* **Rejected Alternatives** :
  - Vendre extended-graph + un data.json préconfiguré (1,6 Mo + schéma fragile, écrasé par Obsidian à chaud).
  - Nommer le champ 'title' (défaut du plugin, zéro config) — rejeté pour rester cohérent avec 'titre' des docs projet/epic.
  - Éditer les data.json des plugins en direct — clobber garanti quand Obsidian tourne.
